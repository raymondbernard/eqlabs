Open‑WebUI Mixture of Agents (part 2)

Building multi‑agent workflows and evaluation pathways.

Youtube Video : https://youtu.be/KxT7lHaPDJ4

Introduction

The Mixture‑of‑Agents (MoA) methodology has demonstrated state‑of‑the‑art performance using open‑source models. We created two pipelines (Groq and Ollama) for Open‑WebUI. These pipelines serve as UI‑agnostic, OpenAI‑compatible plugin frameworks.

Open‑WebUI is an extensible, feature‑rich, and user‑friendly self‑hosted WebUI designed to operate entirely offline. It supports various LLM runners, including Ollama and OpenAI‑compatible APIs. Here we integrate MoA into this platform and connect evaluation hooks to EQUATOR for deterministic checks (correctness, completeness, groundedness).

Install Open‑WebUI

# Windows
py -m venv .venv
.venv\Scripts\activate

git clone https://github.com/open-webui/open-webui.git
cd open-webui
pip install -r requirements.txt
open-webui serve  # then visit http://localhost:8080

Install Pipelines

git clone https://github.com/open-webui/pipelines.git
cd pipelines
pip install -r requirements.txt

# Create .env (use your own keys)
# GROQ_API_BASE_1=https://api.groq.com/openai/v1
# GROQ_API_KEY_1=<your key>

# Windows
start.bat

# Linux
sh start.sh

Configure MoA (Groq)

In Open‑WebUI admin settings, install the MoA Groq pipeline via the GitHub URL. Configure models/keys in .env. The pipeline fans out your prompt to three reference models, aggregates responses, then synthesizes a final answer with an aggregator model.

Key components to verify

  1. Send the prompt to 3 reference models
  2. Aggregate the responses
  3. Use an aggregator agent to synthesize the final response

Evaluation hooks feed into EQUATOR so you can log per‑criterion outcomes for auditability.
